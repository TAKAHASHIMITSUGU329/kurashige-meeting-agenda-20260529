# PDFリネーム実証 不動産管理資料 初回

## 目的

スキャンPDFや画像PDFの内容をAIで読み取り、手作業で開かなくても内容が分かるファイル名へ自動リネームできるかを検証する。

蔵重税理士事務所さまの課題である「紙書類スキャン時のファイル名自動付与・NAS保存」の初回実証として実施した。

## 対象ファイル

| 元ファイル | ページ数 | 内容 |
| --- | ---: | --- |
| `2026.04.pdf` | 4ページ | 不動産管理精算書 |
| `2026.04カノークス.pdf` | 3ページ | 送金の御案内・送金明細 |
| `2026.04鶴原.pdf` | 6ページ | 月次収支報告書・入居者別明細 |

## 実証結果

PDFは埋め込みテキストがほぼなく、通常のテキスト抽出では内容を取得できなかった。

そのため、PDFをページ画像として確認し、帳票上の以下の情報を読み取ってファイル名に反映した。

- 対象年月
- 宛先または関係先
- 物件名
- 帳票種別
- 主要金額

## 命名ルール

今回の実証では、以下の形式で命名した。

```text
YYYY年MM月_宛先または顧客名_物件名_帳票種別_主要金額.pdf
```

例：

```text
2026年04月_フォルテ_カノークスツリーSマンション_送金案内_振込683571円.pdf
```

## 今回確認できたこと

- 画像PDFであっても、ページ画像をAIで確認すれば帳票種別・物件名・対象年月・金額を読み取れる。
- ファイル名に物件名と対象年月を入れることで、一覧表示だけで内容を判別しやすくなる。
- 元ファイルを残したまま、説明的なファイル名のコピーを作成できる。
- 不動産管理票、送金案内、月次収支報告書のような帳票は、自動リネームの対象として相性がよい。

## 実務運用イメージ

1. 複合機またはスキャナで紙書類をPDF化する。
2. スキャンPDFが一時フォルダに保存される。
3. AIがPDFの1ページ目または全ページを画像として読み取る。
4. 対象年月、物件名、帳票種別、金額を抽出する。
5. 命名ルールに沿ってファイル名を作成する。
6. 元ファイルを残すか、リネーム後ファイルだけ残すかを運用ルールで決める。
7. 物件名または顧客名ごとにNASの正しいフォルダへ保存する。

## 自動化時の出力項目案

AIがPDFごとに以下のJSON形式で情報を返すと、後続のリネーム・保存処理に使いやすい。

```json
{
  "target_year_month": "2026年04月",
  "client_name": "フォルテ",
  "property_name": "カノークスツリーSマンション",
  "document_type": "送金案内",
  "main_amount_label": "振込",
  "main_amount": 683571,
  "suggested_filename": "2026年04月_フォルテ_カノークスツリーSマンション_送金案内_振込683571円.pdf",
  "confidence": "high",
  "needs_human_review": false
}
```

## 注意点

- スキャン品質が低い場合、物件名や金額の読み取りミスが起きる可能性がある。
- 複数物件が1つのPDFに含まれる場合、どの物件名をファイル名に採用するかルールが必要。
- 金額が複数ある帳票では、「振込額」「収支合計」「物件支払額」など、どの金額を主要金額にするか決める必要がある。
- 自動保存まで行う場合、NAS側のフォルダ体系と物件名マスタが必要。
- 初期運用では、AIが作ったファイル名を人が確認してから確定するのが安全。

## 次回検証したいこと

- スキャン直後の生PDFを追加で受領し、読み取り精度を確認する。
- 複数形式の不動産管理資料に対して、同じ命名ルールで処理できるか試す。
- 物件名とNAS保存先フォルダを対応させるマスタを作る。
- 自動リネームだけでなく、フォルダ振り分けまで実証する。
- 読み取り結果に自信がない場合は、`要確認_` をファイル名の先頭につける運用を検討する。
