## 今回確認できたこと

- AIに対して「フォルダを理解して」「資料内容を読んで」「成果物としてMarkdown化して」と依頼すると、作業ログと成果物を残しながら進められる。
- 仕訳チェックは、先にチェック観点をMarkdownで定義すると、結果の粒度を揃えやすい。
- PDFリネームは、画像PDFでも内容理解と命名案作成が可能。実運用では保存先マスタと例外処理が重要になる。
- 財務分析は、試算表だけでも面談用の論点整理は可能。ただし前期比較、月次推移、借入返済予定などがあると実務精度が上がる。
- GitHub PagesにHTMLとMarkdownを置くことで、打ち合わせ時に成果物をリンクで確認できる。

## 次回以降の指示テンプレート

### 仕訳チェック

```text
このCSVを読み取って、事務所のチェック観点に沿って確認候補を出してください。
チェック観点、チェック結果、追加で確認すべき事項をMarkdownでまとめてください。
```

### PDFリネーム

```text
このPDFの内容を読み取り、対象年月、顧客名、物件名、帳票種別、主要金額を抽出してください。
元ファイルは残したまま、分かりやすいファイル名でコピーしてください。
実証結果をMarkdownにまとめてください。
```

### 財務分析

```text
この試算表を読み取り、顧客面談で使える一言サマリー、良い点、気になる点、質問候補をMarkdownでまとめてください。
数字の根拠も併記してください。
```

### 打ち合わせ資料化

```text
作成したMarkdownやPoC資料をリンクできるHTMLハブを作成し、GitHub Pagesにプッシュして表示してください。
```
