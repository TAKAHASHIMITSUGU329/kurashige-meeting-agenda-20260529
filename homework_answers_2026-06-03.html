結論

Claude Skills 利用可能。Free/Pro/Max/Team/Enterpriseで使え、Code execution and file creationの有効化が前提。会計チェック観点やPDF命名規則は「カスタムSkill化」に向く。
クォータ制限 ロックは故障というより、5時間枠・週次枠・作業量ベースの消費に当たっている可能性が高い。大きいPDF・長い会話・高推論モデルは消費が重い。

1. Claude Skillsの回答

答え

Claude Skillsはあります。Claudeに専門知識・手順・会社固有のワークフローを持たせるための機能で、Excel、Word、PowerPoint、PDFなどのAnthropic製Built-in Skillsも用意されています。カスタムSkillはZIPでアップロードできます。

事務所での使いどころ

  • 仕訳チェックSkill: 弥生CSVを読み、科目・税区分・給与・手形・大口取引を確認する。
  • PDF命名Skill: 不動産管理PDFから年月・物件・帳票種別・金額を読み、命名規則に従って候補名を出す。
  • 財務分析Skill: 試算表から面談用サマリー、良い点、気になる点、社長への質問を出す。
  • 給与計算補助Skill: 社保料率表・年齢・都道府県・報酬月額をもとに計算根拠を示す。
根拠

Anthropic Help Centerでは、SkillsはFree/Pro/Max/Team/Enterpriseで利用可能、Team/Enterpriseでは組織配布も可能と説明されています。Built-in SkillsにはExcel、Word、PowerPoint、PDF関連が含まれます。Use skills in Claude / What are Skills?

2. クォータロックの回答

答え

ロック原因は、単純な「回数制限」ではなく、会話の長さ、添付ファイルサイズ、PDF枚数、ツール利用、モデルの重さが合算されて制限に達した可能性が高いです。Google Antigravityも、作業量が多いタスクほどクォータを消費します。

確認すべきこと

  • ロック時の画面表示: 「5時間後に回復」か「週次回復」か。
  • 使用モデル: Gemini Pro系、Claude Opus系など重いモデルだったか。
  • 投入物: PDFページ数、画像PDFかテキストPDFか、会話が長くなっていたか。
  • 同じ共有アカウントを複数人で使っていないか。
実務上の見立て

「最長7日リフレッシュされない」は、週次枠または無料枠/大きい作業量枠に当たった可能性があります。Pro/Ultraの5時間更新枠でも、重いPDF・長い会話・高推論モデルを使うと短時間で消費します。

根拠

Claudeはメッセージ長、ファイルサイズ、会話長、ツール利用が利用制限に影響します。Claude Codeでは`/usage`でリセット時刻を確認できます。Google AntigravityはPro/Ultraで5時間ごとに更新されるクォータがあり、作業量に応じて消費されます。Claude usage limit best practices / Claude Code error reference / Google Antigravity rate limits

3. PDFトークン消費の回答

答え

スキャンPDF処理は重いです。ClaudeではPDFページを画像化し、抽出テキストと画像の両方を処理します。APIの目安では、テキスト分だけで1ページあたり約1,500-3,000 tokens、さらに画像処理分が加わります。

ざっくり試算

PDF 想定消費 実務判断
1ページの送金案内 Claude: 少なくとも1,500-3,000 tokens + 画像分 リネーム用途なら許容。まずは1-5件単位で運用。
3ページの不動産精算書 Claude: テキスト分だけで4,500-9,000 tokens + 画像分 一括大量投入は避ける。物件ごと・日ごとに分割。
画像PDF 20ページ 高負荷。会話履歴も長いとすぐ制限に近づく 必要ページだけ抜き出す。OCR済みテキスト化も検討。

消費を抑える運用

  • リネームだけなら「対象年月、物件名、帳票種別、金額」だけ抽出させる。
  • 命名規則は毎回チャットで長く書かず、MarkdownまたはSkillに固定する。
  • 同じPDFを何度も読ませない。結果をMarkdown/CSVに保存する。
  • 大きいPDFはページ分割する。画質が悪いスキャンは先に回転・補正する。
  • 会話が長くなったら新しいチャット/新しいプロジェクトに切る。
根拠

ClaudeのPDFサポートでは、PDFページを画像化し、テキストと画像として処理します。Gemini APIもPDFを視覚的に理解し、Gemini 3ではPDFページ由来の画像処理トークンを`IMAGE`として報告します。Claude PDF support / Gemini document understanding

4. 次回までの推奨運用

PDFリネーム 1回の投入は1-5ファイル程度。命名規則を固定し、結果をMarkdownまたはCSVに保存。大量処理は後で自動化。
仕訳チェック 観点リストをMarkdownで固定。AIには「観点に照らして、疑義・理由・確認質問を表で出す」と指示。
財務分析 Excel/試算表をもとに、社長面談用の一言サマリー、確認事項、質問案だけを作る。

5. LINE共有用メッセージ案

本日の宿題について調べたところ、ClaudeのSkills機能は利用可能でした。会計チェック観点やPDF命名規則を「Skill」やMarkdownとして持たせると、毎回同じ品質でAIに作業させやすくなります。 一方で、PDF読み取りはかなりトークン消費が大きく、特にスキャンPDFは制限に当たりやすいです。まずは1-5件程度の小さいPDFで、命名規則を固定して試すのが安全です。 次回までに「AIにやらせたいこと」「対象ファイル」「理想の出力」を送っていただければ、こちらで事前に実証して共有します。
